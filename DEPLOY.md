# Запуск Telegram-бота 24/7 на PythonAnywhere

Эта инструкция нужна, чтобы бот работал круглосуточно — даже когда ваш компьютер выключен.

---

## Коротко

| Что | Где | Чем |
|-----|-----|-----|
| Хостинг | pythonanywhere.com | Тариф **Hacker** ($5/мес — обязателен для Always-On) |
| Файлы для загрузки | папка `tgbot/` на сервере | `main.py`, `config.json`, `requirements.txt`, `secrets.env` |
| Запуск | Always-on task | `python3 /home/USERNAME/tgbot/main.py` |

> **Важно про бесплатный тариф**: на нём нет «Always-On task». Ваш бот будет засыпать через несколько часов. Для 24/7 нужен тариф Hacker минимум.

---

## Шаг 1 — Регистрация и оплата

1. Откройте https://www.pythonanywhere.com и нажмите **Sign up**
2. Подтвердите email, войдите в аккаунт
3. Перейдите в **Account** → **Upgrade** → выберите тариф **Hacker** ($5/месяц)
4. Запомните ваш `username` — он показан в правом верхнем углу. Дальше в инструкции я пишу `USERNAME` вместо вашего реального.

---

## Шаг 2 — Создать папку на сервере

1. Dashboard → **Files**
2. В адресной строке создайте новую папку: впишите `tgbot/` после имени `home/USERNAME/` и нажмите **New directory**
3. Войдите внутрь папки `tgbot/`

---

## Шаг 3 — Загрузить файлы бота

В папке `tgbot/` нажмите **Upload a file** и загрузите по очереди эти файлы из вашей локальной папки `C:\Users\serge\Desktop\TelegramBot\`:

- `main.py`
- `config.json`
- `requirements.txt`
- `secrets.env`

> **Не загружайте**: `__pycache__/`, `.gitignore`, `index.html`, `DEPLOY.md` — они не нужны для работы бота.

---

## Шаг 4 — Установить библиотеки

1. Dashboard → **Consoles** → **Start a new console** → **Bash**
2. Выполните:
   ```
   cd ~/tgbot
   pip install --user -r requirements.txt
   ```
3. Дождитесь сообщения `Successfully installed pyTelegramBotAPI-...`

---

## Шаг 5 — Проверить, что secrets.env загружен

В той же Bash-консоли:
```
ls -la secrets.env
```

Должно показать примерно: `-rw-r--r-- 1 USERNAME USERNAME 80 May 1 secrets.env` (размер ~70–90 байт).

> **Содержимое токена через `cat` не печатайте** — лог консоли может попасть на скриншот. Если нужно отредактировать — откройте файл через интерфейс **Files** в браузере (там удобный редактор).

Если в файле всё ещё `BOT_TOKEN=PASTE_TOKEN_HERE` — откройте его через Files, замените на реальный токен от `@BotFather`, нажмите **Save**.

---

## Шаг 6 — Тестовый запуск (1 минута)

В Bash-консоли:
```
cd ~/tgbot
python3 main.py
```

Должно вывестись:
```
Бот запущен (Название бизнеса). Останов: Ctrl+C
```

Откройте Telegram → найдите своего бота → нажмите `/start` → должны появиться кнопки. Если работает — нажмите **Ctrl+C** в консоли, чтобы остановить тест.

> Если выскочила ошибка про токен или конфиг — исправьте сейчас. **Не переходите к шагу 7 пока тест не прошёл.**

---

## Шаг 7 — Always-On Task (запуск 24/7)

1. Dashboard → **Tasks**
2. Прокрутите вниз до раздела **Always-on tasks**
3. Нажмите **Create a new always-on task**
4. В поле **Command** впишите (замените `USERNAME` на ваш):
   ```
   python3 /home/USERNAME/tgbot/main.py
   ```
5. **Description** (опционально): `Telegram bot`
6. **Create**

Через 10–30 секунд статус задачи станет **Running** (зелёная точка).

---

## Шаг 8 — Проверить работу

1. На странице **Tasks** → **Always-on tasks** статус задачи = **Running**
2. В Telegram → бот → `/start` → бот отвечает кнопками
3. Закройте все консоли. Закройте браузер. Выключите компьютер. Через час включите и снова напишите боту — он должен продолжать отвечать.

## Управление ботом

### Посмотреть лог
Tasks → Always-on tasks → клик по ссылке-логу рядом с задачей. Видно `print()` из бота, в том числе уведомления о принятых заявках.

### Перезапустить (после правки кода)
1. Загрузите новый `main.py` через Files (заменит старый)
2. Tasks → нажмите **Restart** на задаче
3. Через 10–30 секунд бот работает с новым кодом

### Остановить
Tasks → **Stop** на задаче. Бот выключится, на запросы отвечать не будет, пока не нажмёте Start.

### Изменить услуги/цены/адрес
Через Telegram (кнопка ⚙️ Панель владельца) — изменения сохраняются в `config.json` на сервере PythonAnywhere, переживают любые перезапуски.

## Безопасность

- `secrets.env` лежит **только на сервере PythonAnywhere**, в публичный доступ не попадает
- В коде `main.py` токена **нет** — он читается из `secrets.env` при старте
- `.gitignore` в локальной папке защищает от случайной загрузки секретов в git, если позже добавите версию контроля
- **Никогда** не пересылайте `secrets.env` через мессенджеры/email/чаты
- **Если случайно показали токен скриншотом** — немедленно идите в `@BotFather` → команда `/revoke` → выпустит новый токен, старый перестанет работать. Замените `secrets.env` на сервере и нажмите Restart на задаче.

## Частые проблемы

| Симптом | Решение |
|---------|---------|
| Always-on task сразу падает в **Failed** | Откройте лог задачи, прочтите ошибку. Чаще всего — `BOT_TOKEN` пустой/неверный или нет `pyTelegramBotAPI` |
| `ModuleNotFoundError: No module named 'telebot'` | Не выполнен Шаг 4 — `pip install --user -r requirements.txt` |
| `Token must contain a colon` | В `secrets.env` нет/неверный токен — отредактируйте через Files |
| Бот молчит, но Always-on Running | Вы написали боту впервые? Telegram кэширует /start — нажмите снова. Или проверьте, что username бота правильный |
| Заявки не приходят владельцу | В `config.json` `admin_chat_id` = 0 или не ваш chat_id. Откройте файл через Files и впишите ваш chat_id (узнать у `@userinfobot`) |
| После перезапуска `config.json` сбросился | Не должен. Если сбросился — значит был залит старый файл. Скачайте текущий `config.json` через Files перед перезаписью с локального компьютера |
